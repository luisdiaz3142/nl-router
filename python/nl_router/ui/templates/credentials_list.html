Credentials

Encrypted credentials referenced by destinations. The UI shows only metadata — name, kind, last-used, expires-at. Plaintext payloads are never returned to any client; they live AES-256-GCM-encrypted in the database and are decrypted in-process by the dispatcher.

Create + rotate credentials via the CLI: nl-router credential create, nl-router credential rotate. A per-kind UI form (basic_http, bearer_token, api_key, aws_keys, gcp_service_account, mtls_cert, tls_cert) is on the M14 follow-up list.
